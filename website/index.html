System Workflow

1

Task Submission

Client submits tasks to the Master node, which stores them in the task queue and prepares worker snapshot.

Client ‚Üí Master: SubmitTask(task) Master ‚Üí TaskQueue: Enqueue(task) Master ‚Üí WorkerRegistry: GetSnapshotOfWorkers()
2

AI Planning

Master requests optimal plan from Planner service using current tasks and worker availability.

Master ‚Üí Planner: RequestPlan(tasks, workers) Planner: Compute optimal assignment (A*, OR-Tools) Planner ‚Üí Master: Return PlanResponse
3

Task Dispatch

Master dispatches tasks to assigned workers, which launch them in containers or VMs.

Master ‚Üí Worker: AssignTask(task) Worker ‚Üí ContainerManager: LaunchTask(task)
4

Execution & Monitoring

Workers execute tasks, send heartbeats, and report completion or failures back to Master.

Worker ‚Üí Master: Heartbeat(status, resources) Worker ‚Üí Master: ReportTaskCompletion(task, status)
5

Failure Handling & Replanning

Monitor detects failures, triggers replanning, and reassigns failed tasks to healthy workers.

Monitor ‚Üí Master: DetectFailure(workerX) Master ‚Üí Planner: Replan(failedTasks, availableWorkers) Master ‚Üí Workers: ReassignTasks(failedTasks)
6

Continuous Operation

System continuously accepts new tasks, plans, executes, and adapts to changing conditions.

Loop: New tasks ‚Üí Plan ‚Üí Execute ‚Üí Monitor
