// Step 1: Component mounts
useEffect(() => {
  // Step 2: Connect to WebSocket
  wsManager.connect('/ws/telemetry', handleMessage);
  
  // Step 3: Receive data
  function handleMessage(data) {
    // data = { "worker-1": { cpu_usage: 45.2, ... } }

  // Step 4: Cleanup on unmount
  return () => wsManager.disconnect('/ws/telemetry');

## üéì Key Learnings & Best Practices

### 1. WebSocket Management
‚úÖ **DO:**
- Use a centralized WebSocket manager
- Implement auto-reconnect logic
- Clean up connections on unmount
- Handle connection errors gracefully

‚ùå **DON'T:**
- Create multiple WebSocket connections for same endpoint
- Forget to close connections
- Block UI thread with WebSocket logic

### 2. State Management
‚úÖ **DO:**
- Use custom hooks for shared logic
- Keep state close to where it's used
- Update state immutably
- Use useEffect cleanup functions

‚ùå **DON'T:**
- Prop drill through many levels
- Mutate state directly
- Create unnecessary global state

### 3. Performance
‚úÖ **DO:**
- Use React.memo for expensive components
- Implement proper useEffect dependencies
- Debounce search inputs
- Lazy load route components

‚ùå **DON'T:**
- Re-render entire lists on every update
- Fetch data in render functions
- Create new objects/arrays in render
